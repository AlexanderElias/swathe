Binders | Attribute Names

Attribute names or Binders are used for Swathe to detect elements to interact with. There are special Binders listed beneath which have special actions. You are not limited to these though you have full access to map directly to any JavaScript property on an Element. The mapping pattern except for special Binders will convert - or dashes to camel case and leave . dots unchanged. For example s-this-style.background is an acceptable syntax.

  • s-[*] The primary prefix for attribute names.
  • data-s-[*] The secondary prefix for attribute names.

Adapters | Attribute Values

Attribute values or Adapters are used to access the Model. The attribute name s-text could have a Adapter set to array.0, array[0], this would then assign the value of the Adapter to the elements innerText.

  • . Dot syntax can be used to access any property/array on the model.
  • [] Bracket syntax can also be used.

Special Binders

  • s-if Shows or hides the element if the Adapter is false || true can be a path on the model or simple false or true.
  • s-class-[name] Adds or removes the element if the Adapter is false || true can be a path on the model or simple false or true.
  • s-text Maps to innerText
  • s-html Maps to innerHtml
    • Acceptable values are an html string or a local path for an Ajax file request.
  • s-style Parses a single level deep object with key value pairs to css properties.
  • s-on-[event name] A browser event. Example s-on-click="clicked(model.value, 'string', 1, true)".
    • Function parameters can be Model Properties, String, Number, and Boolean. Numbers and Booleans are parsed to the native type. (Currently strings can not have commas)
    • The last arguemnt is the Event that was fired.
  • s-for-[variable name] A for of loop. Example s-for-index="iterable".
    • for Clones the first child element.
    • variable The value for the children elements to use.
    • index The key or index of an array/iterable on the model.
