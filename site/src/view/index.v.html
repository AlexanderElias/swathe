Attribute Names

Attribute names are used for Swathe to detect elements to interact with. There are special Swathe attribute names listed at the bottom which have special actions. You are not limited to just these special attributes, you have full access to map directly to any JavaScript DOM property.

Since camel cased string are not an acceptable format for browsers the - will be converted to camel case. And then mapped to a JavaScript DOM property.

Infinintly deep object paths are also allowed in attribute names for example s-style.background-color="blue" will set the elements background color to blue.

  • s-[*] The primary prefix for attribute names.
  • data-s-[*] The secondary prefix for attribute names.

Attribute Values

Attribute values are used to access the Model and assign values to the view. An attribute name of s-text could have the value of array[0] this would then assign the value from the model to the inner text of the element.

In most case if the attribute value is not found on the Model the value will attempt to convert to a Primitive JavaScript type.

  • . Dot syntax can be used to access any property including arrays on the model.
  • [] Bracket syntax can also be used. But it will be converted to dot syntax.

Special Attribute Names

  • s-check | s-uncheck
  • s-show | s-hide
  • s-class-[class name]
  • s-for-[variable name]
  • s-on-[event name]
  • s-html
  • s-css
